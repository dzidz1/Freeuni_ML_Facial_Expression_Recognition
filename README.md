# Facial Expression Recognition Challenge

კონკურსის აღწერა: კონკურსის მიზანია ისეთი მოდელის ტრენინგი, რომელიც სურათზე ამოიცნობს 7 ემოციიდან ერთ-ერთს (0=Angry, 1=Disgust, 2=Fear, 3=Happy, 4=Sad, 5=Surprise, 6=Neutral). მონაცემები 48x48 ზომის grayscale სახეებია, პიქსელები კი CSV-ში space-ით გამოყოფილი სტრინგის სახით ინახება.

მონაცემებად ავიღე icml_face_data.csv, რომელსაც აქვს Usage სვეტი, ოფიციალური დაყოფა Training / PublicTest / PrivateTest-ად (28709 / 3589 / 3589). აღნიშნული დატასეტი შევარჩიე, რათა შედეგები ყოფილიყო ფაბლიქ ბენჩმარკთან შედარებადი, მქონოდა ლეიბლირებული ტესტ სეტი და ჩემით არ დამეყო ტრეინ სეტი.

ემოციების გადანაწილება თითქმის თანაბარია, გარდა disgust-ისა, რომელზეც მხოლოდ 547 მაგალითია. შედარებისთვის happy 8989-ჯერ გვხვდება (დაახლოებით 16-ჯერ მეტი). ეს დისბალანსი FER2013-ის მთავარი სირთულეა და ნამდვილად აისახა შედეგებშიც. პიქსელებს გავუკეთე ნორმალიცაზია - მნიშვნელობები [0, 255] გადავიყვანე [0, 1]-ში (გავყავი 255-ზე), batch size-ად ავიღე 64. მონაცემები ჩავსვი ცალკე Dataset კლასში, რომ მოგვიანებით augmentation მარტივად შემძლებოდა.

სანამ მოდელის ტრენინგს დავიწყებდი შევასრულე sanity check-ები forward-ის და backward-ის. თავდაპირველად შევამოწმე საწყისი loss, ტრენინგის გარეშე მოდელი შვიდივე ემოციას დაახლოებით თანაბრად უნდა ხვდებოდეს, ანუ loss უნდა იყოს -ln(1/7) = 1.946-თან ახლოს. მივიღე 1.97 ანუ output ლეიერი და loss სწორად არიან შეერთებული და არ მაქვს შემთხვევითი დუბლირებული softmax ან მსგავსი შეცდომები. ამის შემდეგ ერთი batch-ისთვის ვცადე overfit. მხოლოდ 64 სურათზე დავატრენინგე, და მე-80 step-ზე accuracy-მ 100%-ს მიაღწია, რაც ადასტურებს, რომ backprop და optimizer ნამდვილად სწავლობენ. ორივე შემოწმებამ წარმატებით გაიარა, ანუ პაიპლაინი გამართულია. სწორედ ამის შემდეგ დავიწყე ტრენინგის პროცესი.

თავდაპირველად მარტივი CNN მოდელი გავაკეთე, 2 convolutional ლეიერით, თითოეულთან ReLU და pooling გამოვიყენე, ბოლოს flattening, loss-ად crossEntropy და ოპტიმიზაციის ალგორითმად Adam შევარჩიე. ამ ეტაპისთვის რეგულარიზაცია არ მქონია. საუკეთესო ვალიდაცია მომცა 53.2%, მაგრამ დაფიქსირდა საინტერესო ფაქტი, train accuracy-მ 98%-ს გადააჭარბა, ვალიდაციის loss კი მე-6 ეპოქის შემდეგ 1.3-დან 4.1-ზე ავიდა. აღნიშნული კლასიკური overfit-ის მაგალითია - დიდი capacity და ნულოვანი რეგულარიზაცია, რის გამოც მოდელმა დაიზეპირა ტრეინ სეტი. სწორედ ეს არის baseline ჩემს პროექტში.

შემდგომ უფრო ღრმა CNN გავაკეთე - 3 conv ბლოკი, თითო Conv->BatchNorm->ReLU->MaxPool პლუს dropout კლასიფაიერი. ერთსა და იმავე არქიტექტურაზე რეგულარიზაციის პარამეტრების ცვლილებით დავინახე მთელი bias-variance სპექტრი. რეგულარიზაციის გარეშე - train 97.5%, val 59.1% (სხვაობა დაახლოებით 38%), მაშასადამე, overfit-შია. dropout 0.5-ით და weight decay 1e-4-ით — train 62.7%, val 56.6% (სხვაობა დაახლოებით 6%, დაბალანსებულია). dropout 0.8-ით და weight decay 1e-2-ით — train 53.6%, val 54.9% შედეგები მომცა (ორივე დაბალი), ანუ მოდელი ისე შევზღუდე, რომ ტრეინიც ვეღარ ისწავლა. შემდეგ ვცადე augmentation (dropout 0.3 ამ ეტაპისთვის), შედეგები ასე გამოიყურებოდა: train 63.2%, val 63.2% (სხვაობა 0). აქედან გამომდინარე გამოვიტანე დასკვნა, რომ train-ზე accuracy მონოტონურად ეცემა რეგულარიზაციის ზრდასთან ერთად. ასევე აღსანიშნავია, რომ augmentation-მა აჯობა dropout-ს. augmentation-მა სხვაობა accuracy-ებს შორის შეამცირა ტრეინის accuracy-ის შემცირებით, augmentation-მა კი accuracy გაზარდა და სხვაობაც შეამცირა.

ამ ყველაფრის შემდეგ გადავედი ResNet-ზე. clean მონაცემებზე (augmentation-ის გარეშე) dropout 0.3-ით მივიღე 57.2% (DeeperCNN-ს ოდნავ აჯობა, მაგრამ სწრაფად overfited გახდა). augmentation-ის დამატებით — 63.3%, ანუ DeeperCNN-ს გაუტოლდა, რაც მაჩვენებს, რომ ამოცანა from-scratch მოდელებისთვის არის data-limited და არა - architecture-limited. უკეთესი შედეგისთვის დავამატე learning rate scheduler (ReduceLROnPlateau, factor 0.5, patience 3) და მივიღე 66.7%. ეს გამოდგა საუკეთესო მოდელი, რომელიც დავატრენინგე from-scratch. scheduler ის საშუალებით ვალიდაცია ყოველი LR დაცემის მერე ოდნავ უმჯობესდებოდა.

გადავწყვიტე კიდევ მეცადა რაიმე და მივედი transfer learning-მდე. გამოვიყენე ImageNet-ზე წინასწარ დატრენინგებული ResNet18. მონაცემები მოვარგე მოდელს: თითო 48x48 სურათი გავადიდე 96x96-მდე, ერთი ფილტრი 3-ად გავამრავლე და ImageNet-ის სტატისტიკით დავანორმალე, ბოლო 1000-კლასიანი head კი 7-კლასიანით ჩავანაცვლე. სამი სტრატეგია შევადარე ერთმანეთს. frozen backbone-ით მივიღე მხოლოდ 39.1%, frozen ImageNet feature-ები grayscale სახეებს კარგად ვერ ჰყოფს. layer3+layer4-ის unfreeze-ით 66.4%-იან შედეგზე გავედი, ხოლო სრული fine-tuning-ით - 69.1%. სწორედ ეს აღმოჩნდა საუკეთესო მოდელი. სრულმა fine-tuning-მა პირველი ეპოქის მერევე 55.7% მომცა, ანუ უკვე აჯობა სრულად დატრენინგებულ SimpleCNN-ს, რადგან ImageNet-ის feature-ები მაშინვე გამოდგა. საინტერესოა ციფრებიც: frozen-დან full-მდე სხვაობა 30%-ია და მხოლოდ ბოლო ორი ბლოკის unfreeze-მა ამ სხვაობის 27% დააბრუნა, ხოლო ადრეული ლეიერების unfreeze-მა მხოლოდ 2.7% მოამატა. ეს ხაზს უსვამს იმ ფაქტს, რომ ბოლო ლეიერები აკეთებენ თითქმის მთელ task-specific საქმეს და ადრეული ლეიერები უფრო ზოგადია.

საბოლოო შედეგების შეჯამება: SimpleCNN baseline — 53.2%, DeeperCNN augmented — 63.2%, ResNet (scratch) augmented + scheduler — 66.7%, ResNet18 (pretrained) full fine-tuning — 69.1%.

ბოლოს საუკეთესო მოდელი ერთხელ გავუშვი ტესტ სეტზე (PrivateTest, 3589 სურათი), რომელიც ტრენინგის დროს არასდროს მინახავს. ტესტ accuracy: 69.5% მივიღ, რაც არის ოდნავ მაღალი ვალიდაციაზე (69.1%), ანუ მოდელი ნამდვილად განზოგადებს და ვალიდაციაზე არ მომხდარა overfit. კლასების მიხედვით: Happy-ს ჰქონდა საუკეთესო recall (0.89). ყველაზე რთული Fear აღმოჩნდა (recall 0.40) — შეცდომები იფანტება Sad-ში (0.20), Angry-ში (0.13), Surprise-ში (0.13) და Neutral-ში (0.12), რადგან შიშის გამომეტყველება რამდენიმე ემოციას ჰგავს.
